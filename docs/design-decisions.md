# Design Decisions

## Purpose
This document records important design decisions as they are made, along with the reasons for each decision.

---

## Decision Log

### DD-001
**Decision:** Use the repository primarily as a documentation and planning space during the early project phase.  
**Reason:** The project is still in concept definition, and organizing notes first will make later engineering work cleaner.

### DD-002
**Decision:** Use a PLC-centered control concept in the documentation.  
**Reason:** The client specifically mentioned using a programmable logic controller, and PLCs are a strong fit for industrial machine control.

### DD-003
**Decision:** Include a local HMI as part of the base concept.  
**Reason:** The client requested adjustable parameters and operator visibility.

### DD-004
**Decision:** Keep remote features in the “future / optional” category for now.  
**Reason:** Remote functionality adds complexity, security concerns, and safety considerations that should not define the base machine concept yet.

### DD-005
**Decision:** Organize notes into separate docs by discipline (controls, hydraulic, mechanical, instrumentation).  
**Reason:** Makes the project easier to expand later without losing information.
